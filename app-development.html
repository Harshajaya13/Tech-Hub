2. The "Notch" Problem

Move it down

I'll just add `padding-top: 50px` so it looks good on this specific phone.

Safe Area View

I need to wrap the UI in a `SafeAreaProvider` to handle notches on all future devices.

3. The Back Button

→

The Stack

4. The Tunnel

The Error Message

Just show a "Please connect to WiFi" popup.

→

Local Storage

I need to implement local caching (SQLite/Realm) to persist data when offline.

→

5. The Camera

You want to let users upload a profile picture. You click the button, nothing happens.

Blame the Phone

"My phone is buggy. It works on the simulator."

→

The Handshake

I forgot to edit `Info.plist` / `Manifest` to request Camera Permissions from the OS.

→

6. The Keyboard Trap

The user tries to type their password, but the on-screen keyboard pops up and covers the "Submit" button.

The UI Fix

I'll move the button to the top of the screen permanently.

→

Keyboard Avoidance

I need to implement a `KeyboardAvoidingView` that listens to keyboard height events.

→

→

Compliance

Read the strict guidelines, fix the metadata, and resubmit a new build version.

8. The Legacy User

You updated the API, but 20% of your users haven't updated their app in 6 months. The app is crashing for them.

Ignore Them

If they don't update, that's their fault.

→

Backward Compatibility

I need to version my API or implement a "Force Update" modal check on app launch.

Wait... A Discovery!

You prefer the User Experience & Visuals over the strict logic of hardware and operating systems.
